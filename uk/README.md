## Проблема

Ви встановили Hermes Agent. Ви запустили кілька промптів. І що далі?

- **Офіційна документація описує функції — але не показує, як їх комбінувати.** Ви знаєте, що навички існують, але не знаєте, як зв'язати їх з делегацією, голосом та MCP у робочий процес, який реально заощаджує години.
- **Немає чіткого шляху навчання.** Спочатку вивчати MCP чи делегацію? Memory чи personality? Ви в підсумку переглядаєте все і не опановуєте нічого.
- **Приклади занадто прості.** Навичка "hello world" не допоможе вам побудувати робочу систему обслуговування клієнтів, яка використовує messaging gateway, голос та заплановані завдання.

Ви залишаєте 90% потужності Hermes Agent на столі — і не знаєте, чого не знаєте.

## Як Hermes How To Це Вирішує

Це не ще одна довідка про функції. Це **структурований, візуальний, прикладний посібник**, який навчає вас використовувати кожну функцію Hermes Agent з реальними шаблонами, які можна скопіювати у ваш проект сьогодні.

| | Офіційна Документація | Цей Посібник |
|--|------------------------|--------------|
| **Формат** | Довідкова документація | Візуальні підручники з Mermaid діаграмами |
| **Глибина** | Опис функцій | Як це працює всередині |
| **Приклади** | Базові сниппети | Готові до продакшену шаблони |
| **Структура** | За функціями | Прогресивний шлях навчання |
| **Онбординг** | Самостійний | Маршрут з часовими оцінками |
| **Самооцінка** | Відсутня | Інтерактивні тести для виявлення прогалин |

### Що ви отримуєте:

- **14 навчальних модулів**, що охоплюють кожну функцію Hermes Agent — від швидкого старту до контекстних посилань
- **Конфіги для копіювання** — навички, шаблони делегації, MCP конфіги, налаштування голосу, інтеграції з месенджерами та повні пакети плагінів
- **Mermaid діаграми**, що показують, як кожна функція працює всередині, щоб ви розуміли *чому*, а не просто *як*
- **Структурований шлях навчання**, який веде вас від початківця до досвідченого користувача за 11-13 годин
- **Вбудована самооцінка** — проходьте тести після кожного модуля, щоб виявити прогалини

## Як Це Працює

### 1. Визначте свій рівень

Пройдіть тест самооцінки або оберіть свій поточний рівень досвіду. Отримайте персоналізований маршрут на основі того, що ви вже знаєте.

### 2. Дотримуйтесь структурованого шляху

Працюйте через 14 модулів по порядку — кожен будує на попередньому. Копіюйте шаблони безпосередньо у свій проект під час навчання.

### 3. Комбінуйте функції в робочі процеси

Справжня сила — у комбінуванні функцій. Навчіться з'єднувати навички + делегацію + MCP + голос + messaging gateway + cron в автоматизовані пайплайни, які обробляють обслуговування клієнтів, моніторинг та інтеграції.

### 4. Перевірте своє розуміння

Проходьте тести після кожного модуля. Тест вказує на те, що ви пропустили, щоб ви могли швидко заповнити прогалини.

## Початок за 15 Хвилин

```bash
# 1. Клонуйте цей посібник
git clone https://github.com/wchao-hermes/hermes-howto.git
cd hermes-howto

# 2. Скопіюйте свою першу навичку
mkdir -p ~/.hermes/skills
cp -r 03-skills/example-skill ~/.hermes/skills/

# 3. Спробуйте — взаємодійте з навичкою в Hermes Agent

# 4. Готові до більшого? Налаштуйте project memory:
cp 02-memory/project-HERMES.md /шлях/до/вашого/проекту/HERMES.md

# 5. Налаштуйте провайдера:
cp 13-providers/provider-examples/standard-providers.md ~/.hermes/providers/
```

Потрібен повний набір? Ось **налаштування на 1 годину**:

```bash
# Skills (15 хв)
cp -r 03-skills/* ~/.hermes/skills/

# Project memory (15 хв)
cp 02-memory/project-HERMES.md ./HERMES.md

# Delegation шаблони (15 хв)
cp -r 04-delegation/* ~/.hermes/delegation/

# MCP сервери (15 хв)
export GITHUB_TOKEN="***"
hermes mcp add github -- npx -y @modelcontextprotocol/server-github

# Вихідні: додайте voice, messaging gateway, cron, plugins
# Дотримуйтесь навчального шляху для покрокового налаштування
```

## FAQ

**Це безкоштовно?**
Так. MIT ліцензія, безкоштовно назавжди. Використовуйте в особистих проектах, на роботі, у своїй команді — без обмежень, крім збереження повідомлення про ліцензію.

**Це підтримується?**
Активно. Посібник синхронізований з кожним релізом Hermes Agent.

**Чим це відрізняється від офіційної документації?**
Офіційна документація — це довідник функцій. Цей посібник — це підручник з діаграмами, готовими до продакшену шаблонами та прогресивним навчальним шляхом. Вони доповнюють одне одного — почніть тут для навчання, звертайтесь до документації для деталей.

**Скільки часу потрібно на проходження всього?**
11-13 годин для повного шляху. Але ви отримаєте цінність за 15 хвилин — просто скопіюйте шаблон навички та спробуйте.

**Чи можу я зробити внесок?**
Звичайно. Дивіться [CONTRIBUTING.md](../CONTRIBUTING.md) для рекомендацій. Ми вітаємо нові приклади, виправлення помилок, покращення документації та шаблони спільноти.
